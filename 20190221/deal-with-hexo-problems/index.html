Hexo + Github Pages 踩坑记录

Feb 21, 2019 | 折腾 | 811 字 | 3 分钟
条评论

今天利用hexo + Github Pages 建立了中文博客，到能访问为止10分钟搞定，但是后面的换域名，换主题调试花了一个下午。下面把踩过的坑记录下来。

给托管在Github Pages上的hexo博客设定子域名

参考链接：Github Page 自定义子域名

博客托管以后的的地址是 dayuanjiang.github.io，然而我有一个 jiang.jp 的域名，上面有我的英文博客 www.jiang.jp。我自然是想把两个博客的地址联系起来，具体说就是把中文博客的地址从 Github 的地址改成 b.jiang.jp。

要改域名需要3个步骤：

  1. 在域名商的 DNS 设定里增加一条 CNAME 的记录映射到 Github repositor 的地址上。
  2. 在 Github 上把 Custom domian 设定为想用的子域名。
  3. 在 Hexo 的文件夹里增加一个 CNAME 的文件。

下面我具体说一下。

目标

目标是把下面左边的地址换成右边的地址。
xxx 是你 Github 的用户名，我的情况就是 DayuanJiang 。
yyy 就是你想换子域名的主域名，我的情况就是 jiang 。

DNS 设定

我用的是 Godaddy，就是这么设置的。

在 Github 上把 Custom domian 设定为想用的子域名

这个简单。进入你 Github Pages 的仓库，右上角的 Setting 点进去，拉到下面，找到 Custom domain，然后填入自己想改的子域名。

最后在 Hexo 的文件夹上增加一个 CNAME 的文件

找到你 博客 的文件夹，里面有个 Source 的文件夹，点进去，建立一个文件名为 CNAME 的文件（注意不要后缀）。然后用笔记本软件打开后输入你的子域名后保存。最后在命令行里hexo d -g 把修改推送到 Github 上。

我用 Mac 是这么操作的。

然后输入我的子域名。保存。

命令行里hexo d -g 把修改推送到 Github 上。

至此为止该做的都做了，在浏览器输入你的子域名应该能进了。注意 DNS 解析需要花一点时间，没有的话耐心等等。

给文章加分类和标签

先说标签。先新建一个标签页面。

在博客的 source 下会新增一个 tags 文件夹，打开里面的 index.md 的文档。

在下面加入 type 跟 layout 两行然后保存就可以了。 

---
title: tags
date: 2019-02-21 17:12:16
type: "tags"
layout: "tags"
---

最后在主题的文件夹下找到 _config.yml 文件后把里面的 tag 一行注释去掉。

大功告成。categories 和这个一样只不过把每一步的 tag 换成 categories 罢了，这里不再赘述。

本地显示，网页上缺不显示

明明改动或者是文章在本地服务器上测试的时候好好的，Github 服务器上也有，但就是在输入网址后看不到。这种时候下面两种方法都试一遍吧。

  1. 推送到 Github 之前先 hexo clean。
  2. 浏览器 Shift + F5 强制刷新页面。
